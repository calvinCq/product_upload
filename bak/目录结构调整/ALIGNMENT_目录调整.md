# 目录结构调整任务对齐文档

## 原始需求

用户原始需求："utils 和 api 都放到src里面 ,钱多多的相关接口配置也放到.env文件中,调整完请自动跑通代码"

## 需求理解

1. **现状分析**：
   - 当前项目中utils目录位于根目录，需要移动到src目录下
   - 根目录下有api目录，需要移动到src目录下（注意src/api目录已存在）
   - 钱多多API配置可能在代码中硬编码或在其他配置文件中，需要移至.env文件

2. **任务目标**：
   - 将utils目录移动到src目录下
   - 将api目录（包含钱多多API）移动到src目录下
   - 将钱多多API的相关配置迁移到.env文件
   - 更新所有相关文件的导入路径
   - 确保代码能够正常运行

## 边界确认

- **处理范围**：
  - 目录结构调整（utils和api目录）
  - 配置文件迁移（钱多多API配置）
  - 导入路径更新
  - 代码运行验证

- **功能保持**：
  - 所有调整不应改变原有功能
  - 确保调整后的代码可以正常运行

## 技术约束

1. **Python导入机制**：
   - 需要更新所有导入路径
   - 确保模块间依赖关系正确

2. **配置管理**：
   - 敏感信息（API密钥等）必须存储在.env文件中
   - 代码中应使用环境变量获取配置

3. **Windows环境**：
   - 使用Windows PowerShell进行文件操作
   - 注意目录移动时可能的权限问题和覆盖问题

## 项目特性规范

### 功能特性
- 保持原有功能不变
- 改善目录结构，使项目更规范
- 提高配置安全性，将敏感信息移至环境变量

### 非功能特性
- 目录结构符合Python项目最佳实践
- 导入路径规范一致
- 配置管理符合安全最佳实践
