# 教育培训商品详情生成 - 需求对齐文档

## 项目上下文分析

### 现有项目架构
- 项目是一个用于微信小店商品自动生成和上传的系统
- 使用火山大模型API进行图片生成
- 包含商品数据生成、图片生成、商品上传等功能模块
- 支持微信小店API集成

### 技术栈
- Python作为主要开发语言
- RESTful API调用（火山大模型API、微信小店API）
- 环境变量和配置文件管理（.env文件）

## 原始需求
"详情页使用此提示词\"这份文件内容用于设计产品链接主图和详情页，帮我根据微信小店中的【教育培训】类目规则，更新输出无敏感词的版本，可直接用于微信小店中上传商品链接主图和详情页 ,2、根据详情页文案，生成全新内容文案，以下为新链接的具体组成5部分： 
 1、标题： 
 2、老师简介： 
 3、课程大纲： 
 4、适用人群： 
 5、学完收获： 
 不要包含敏感词，【评估、测评、服务、咨询、1v1、改善、高效、最、方案、独家、第一、报告、深度分析、检查、合作洽谈、挑选安心好物】\" ,需要传客户资料进来,同时测试商品添加"

## 需求理解

1. **功能需求**：
   - 创建一个工具，用于生成符合微信小店教育培训类目规则的商品详情
   - 输入：客户资料
   - 输出：5部分组成的商品详情（标题、老师简介、课程大纲、适用人群、学完收获）
   - 确保输出内容不包含指定的敏感词

2. **非功能需求**：
   - 生成的内容必须符合微信小店的审核规则
   - 内容应适合用于商品链接主图和详情页
   - 支持客户资料的输入和处理
   - 能够测试商品添加功能

## 边界确认

1. **任务范围**：
   - 生成教育培训类商品的详情内容
   - 确保内容无敏感词
   - 测试商品添加功能

2. **排除范围**：
   - 不包括其他类目的商品详情生成
   - 不包括图片生成（仅文本内容）
   - 不包括完整的商品上架流程（仅测试添加）

## 疑问澄清

1. 客户资料的具体格式和内容是什么？
2. 生成的内容是否需要与图片生成功能集成？
3. 敏感词列表是否需要扩展或定制？
4. 商品添加测试需要什么具体的验证标准？

## 初步决策

1. 创建一个专门的教育培训商品详情生成器模块
2. 使用现有的商品上传功能进行测试
3. 实现敏感词过滤机制
4. 设计灵活的客户资料输入接口
